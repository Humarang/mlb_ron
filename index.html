Recommended full-data architecture

  1. Frontend: this dashboard for games, teams, players, images, tables, charts, and manual TAB.nz odds.
  2. Backend collector: scheduled jobs that collect schedules, rosters, standings, box scores, game logs, lineups, injuries, and player stats.
  3. Database: PostgreSQL tables for teams, players, seasons, games, lineups, player_game_stats, pitching_stats, batting_stats, injuries, and manual_tab_odds.
  4. Prop engine: store every player game log so you can calculate hit rates for lines such as over/under 4.5, 5.5, 6.5 strikeouts.
  5. TAB.nz: keep manual odds fields unless TAB.nz provides written permission or an official feed.
